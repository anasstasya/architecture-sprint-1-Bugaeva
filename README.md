1. Проектирование
1.1 Домены
В процессе исследования проекта фронтенда были определены следующие домены
1.1.1 Аутентификация и авторизация - домен отвечающий за выполнение входа пользователей в сервис, проверку токенов, а также регистрацию новых пользователей 
1.1.2 Профиль - отвечает за отображение и редактирование информации о пользователях
1.1.3 Места - отвечает за функционал отображения, добавления и удаления фотографий мест в сервис, а также для обработки реакций пользователей на фотографии мест.

1.2 Фреймворк микрофронтендов
Был выбран Webpack Module Federation, так как:
- Наличие только основного роута и роутов для логина и регистрации новых пользователей 
- Требуется композиция из компонентов микрофронтендов, а не полноценные импортированные SPA

1.3 Коммуникационный слой
Для взаимодействия компонентов было решено реализовать событийную модель посредством встроенного механизма событий браузерного движка JS: 
1.3.1 Компоненты доменов публикуют события
1.3.2 Компоненты сервиса композиции потребляют события
1.3.3 Компоненты сервиса композиции передают данные из событий потребителям других доменов, зависящих от данных сведений 
Данный способ коммуникации выбран из за простоты реализации. В дальнейшем можно рассмотреть менеджер глобального состояния mobx

2.1.1 Компоненты:
2.1.1.1 JwtValidateWrapper.js
Обертка для роутов, проверяющая JWT токен при изменении роута.

Содержит логику обработчиков, логику коммуникаций и композицию компонентов страницы входа. При успешном входе или ошибке публикует соответствующее событие.
 
2.1.1.3 LoginForm.js
Содержит верстку формы входа

2.1.1.4 Register.js
Содержит логику обработчиков, логику коммуникаций и композицию компонентов страницы регистрации. 
При успешном входе или ошибке публикует соответствующее событие.
 
2.1.1.4 RegisterForm.js
Содержит верстку формы регистрации.

2.4.1 Компоненты
2.4.1.1 App.js
Содержит маршрутизацию, логику обработчиков, логику обработки событий публикуемых доменами и композицию компонентов страниц, а также всплывающего окна об успешных операциях или ошибках. 
Передает сведения событий компонентам доменов, зависящих от данных других доменов.
2.4.1.2 Footer.js
Футер сервиса.
2.4.1.3 Header.js
Шапка сервиса.
На странице логина отображает кнопку для перехода на страницу регистрации нового пользователя.
На странице регистрации нового пользователя отображает кнопку перехода на страницу входа пользователя.
После входа пользователя отображает электронную почту пользователя и кнопку Выход. По нажатию на кнопку пользователь завершает сеанс и перенаправляется на страницу входа.
2.4.1.4 Main.js
Компонент для композиции компонентов главной страницы: компонента профиля и компонента мест.

3. Запуск:
3.1 Запуск микрофронтенда common-components
3.1.1. В CLI перейти в папку микрофронтенда, выполнив команду из корня проекта:
cd \microfrontends\common-components
3.1.2. Установить зависимости, выполнив команду:

3.1.3. Запустить микрофронтенд, выполнив команду:
